# Plan

作成日時: 2026-05-19 23:05
更新日時: 2026-07-21 10:22

このファイルは、次に進める作業を相談しながら整理するための進捗管理ドキュメントです。

## 現在の優先方針

直近の履歴では、画像ライブラリ、ライブラリ参照チャット、Caption / tags 生成、embedding、FTS5 ハイブリッド検索、UI レイアウト調整が大きく進んでいる。

最近の更新の主軸は、過去画像を保存し、その過去画像を LLM が参照できる材料として蓄積することで、プロンプト改善と目的画像への到達精度を高めることにある。

当面は新しい大機能を増やすより、既存の主要ワークフローを安定させ、日常的に使いやすい状態へ寄せる。

## 短期予定

### 1. 進捗管理ドキュメントの整備

- `docs/plan/goals.md` にプロジェクトの目的と判断基準をまとめる。
- `docs/plan/plan.md` に短期・中期の作業候補をまとめる。
- `docs/plan/progress.md` に完了済み作業と残っている注意点をまとめる。
- `docs/changelog.md` と README の内容が大きく食い違わないようにする。

### 2. 画像ライブラリの安定化

- 画像登録、メタデータ抽出、詳細モーダル、フォルダ移動、複数選択、一括削除の基本操作を確認する。
- Caption / tags 生成の成功時・失敗時の表示を見直す。
- 一括 Caption 生成中の進捗表示とキャンセル・失敗時の扱いを確認する。
- 再インデックス化後の embedding 状態表示をわかりやすくする。
- ライブラリカード、詳細モーダル、サイドバーの操作感を揃える。

### 3. ライブラリ参照チャットの確認

- `{library_context}` を含むシステムプロンプトで、参照件数と検索モードが意図通り効くか確認する。
- ベクトルのみ検索と FTS5 + ベクトル検索の結果差を、実データで確認する。
- Embedding サーバー未起動時や embedding 未作成データが多い場合のフォールバック挙動を確認する。
- プロンプトデバッグビューアで、LLM に渡る文脈量を追いやすくする。
- 過去画像のプロンプト、Caption、tags、notes が、次のプロンプト改善に十分役立つ形で LLM に渡っているか確認する。

### 4. UI の仕上げ

- トップバーのモデル選択、バックエンド操作、VRAM 表示を見直す。
- 画像生成、動画生成、ライブラリ、スニペットの画面密度と余白を揃える。
- 主要ボタンの状態表示、disabled 表示、処理中表示を整理する。
- 小さい画面やウィンドウ幅変更時に、テキストや操作ボタンが崩れないか確認する。

### 5. スニペットの移植

- 旧 Image Assistant（`legacy/`）のスニペット管理機能を Stable Diffusion Studio へ移植する。
- `snippets/` 配下の `.code-snippets` 資産を現行アプリから利用できるようにする。

## 中期予定

### 1. プロンプト管理の強化

- チャット用システムプロンプトのプリセット運用をわかりやすくする。
- Caption 生成プロンプトの編集・複製・削除の扱いを整理する。
- `{positive_prompt}`、`{negative_prompt}`、`{library_context}` などの差し込み変数を UI から把握しやすくする。
- 既定プロンプトとユーザー編集済みプロンプトの関係を壊れにくくする。

### 2. 生成ワークフローの改善

- Forge と ComfyUI の切り替え時に、使える設定と使えない設定をわかりやすくする。
- ComfyUI workflow の差し替え対象ノードを確認しやすくする。
- Seed、サイズ、sampler、steps、CFG の再利用導線を改善する。
- 生成画像をライブラリへ保存する流れをさらに短くする。
- 2 枚の参照画像を読み込み、一方のキャラクター要素ともう一方のポーズ・構図要素を組み合わせてプロンプト化できる導線を検討する。

### 3. 動画生成の使い勝手改善

- 動画プロンプト生成のセクション選択を、実作業に合わせて調整する。
- 動画生成中の進捗、待ち時間、失敗時ログの見え方を改善する。
- 動画 workflow ごとの差し替え項目を整理する。

### 4. 検証手順の明文化

- Python 側の構文確認、主要 API の簡易確認手順をまとめる。
- Electron / frontend 側の起動・検証手順を README または docs にまとめる。
- `package.json` が `electron/` 配下にある構成を前提に、必要な npm コマンドを確認する。
- 手元で完全検証できない外部バックエンド依存部分は、確認観点だけでも残す。

## 継続タスク

- `docs/changelog.md` にユーザー向けの変更を記録する。
- README と実装の差分を見つけたら更新する。
- ユーザーの未コミット変更を戻さず、必要な範囲だけ編集する。
- ログファイルや `data/library/` のようなローカルデータを Git 管理対象にしない。

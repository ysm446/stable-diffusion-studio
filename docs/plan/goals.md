# Goals

作成日時: 2026-05-19 23:05
更新日時: 2026-05-19 23:20

このファイルは、Image Assistant の方向性と判断基準を共有するための進捗管理ドキュメントです。

## 目的

Image Assistant は、ローカル LLM を使って画像生成プロンプトの精度を高め、効率よく目的の絵へ近づくためのデスクトップアプリです。

Stable Diffusion / ComfyUI / ローカル LLM をひとつの作業環境にまとめ、相談、プロンプト修正、生成、結果確認、再利用の反復を速く、見通しよくすることを目指します。

最近の重要な方向性は、過去画像を保存し、その画像に含まれるプロンプト、Caption、タグ、メモ、embedding を参照できる知識として蓄積することです。過去の成功例や近い作例を LLM が参照できるようにし、生成を重ねるほど目的の絵へ近づきやすくします。

中心に置く価値は次の通りです。

- ローカル環境で、LLM 相談・プロンプト改善・画像生成・ライブラリ参照を完結できること。
- 生成結果、既存画像、PNG メタデータ、Caption、タグ、メモを次のプロンプト改善に再利用できること。
- 過去画像を蓄積し、検索・参照・比較しながらプロンプト精度を継続的に高められること。
- Positive / Negative Prompt、Seed、サイズ、workflow、参照画像を扱う作業をなるべく手元の UI だけで済ませ、試行錯誤の回数と手間を減らすこと。
- モデルやバックエンドの起動・解放・接続状態を見える化し、VRAM 制約のある環境でも運用しやすくすること。

## 完成形

当面の完成形は、次の状態を目指す。

- 画像生成タブで、Forge または ComfyUI を切り替えながら txt2img を安定して実行できる。
- 動画タブで、現在画像と動画 workflow を使って ComfyUI の image-to-video 生成を実行できる。
- AI チャットで、現在の画像・プロンプト・ライブラリ参照を使い、生成向けプロンプトを相談・修正できる。
- `[PROMPT_UPDATE]` によるプロンプト自動反映が、通常の作業で安心して使える。
- 画像ライブラリで、画像登録、メタデータ抽出、Caption / tags 生成、embedding、FTS5 + ベクトル検索、フォルダ整理を扱える。
- スニペット管理で、プロンプト素材を検索・編集し、入力補助として使える。
- 主要な設定、モデル選択、バックエンド状態、VRAM 表示が UI 上で把握しやすい。

## 判断基準

実装や改善で迷った場合は、次の基準を優先する。

- 画像・動画生成の実作業が短くなるか。
- 過去の生成資産を、次の生成に再利用しやすくなるか。
- ローカルモデルや生成バックエンドの状態がわかりやすくなるか。
- UI が広くなりすぎず、よく使う操作にすぐ届くか。
- 設定やデータ保存先が、あとから追跡しやすい形になっているか。

## 範囲

優先する範囲:

- Windows ローカル環境での利用。
- `llama-server` の OpenAI 互換 API を使ったローカル LLM。
- SD WebUI Forge と ComfyUI を使った画像生成。
- ComfyUI workflow を使った動画生成。
- `data/library/` 配下のローカル画像ライブラリ。

優先度を下げる範囲:

- クラウド同期や複数端末共有。
- マルチユーザー運用。
- 外部サービス前提の認証・課金・公開機能。
- Web アプリとしての一般公開。
